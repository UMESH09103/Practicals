  * Book a room
  * Cancel a booking

# 🎯 2. Objective (Simple Words)

# 🧠 3. What is RMI?

👉 **RMI = Remote Method Invocation**

✔ It allows:

* A program (client) to call a method on another machine (server)

👉 Simple meaning:

> “Client calls server function like it is local”

---

# 🏨 4. Real-Life Example

* You open a hotel app (client)
* Request booking
* Server checks availability
* Server confirms booking

---

# ⚙️ 5. System Architecture

### 🔹 Components:

1. **Client**

   * Sends request (book/cancel)

2. **Server**

   * Stores room data
   * Processes request

3. **RMI Registry**

   * Connects client and server

4. **Remote Interface**

   * Defines methods

---

# 🔄 6. Working Flow (Step-by-Step)

1. Client sends request
2. RMI sends request to server
3. Server processes request
4. Server returns result
5. Client displays output

# 💻 7. Code Structure (Important)

There are **4 main parts**:

---

## 🔹 1. Remote Interface

```java
import java.rmi.*;
public interface Hotel extends Remote {
    String bookRoom(String name) throws RemoteException;
    String cancelBooking(String name) throws RemoteException;
}
```

### 👉 Explanation:

* Defines methods
* `Remote` → required for RMI
* Methods must throw `RemoteException`

---

## 🔹 2. Server Implementation

```java
import java.rmi.server.UnicastRemoteObject;
import java.rmi.RemoteException;
import java.util.*;

public class HotelImpl extends UnicastRemoteObject implements Hotel {

    List<String> bookings;

    protected HotelImpl() throws RemoteException {
        bookings = new ArrayList<>();
    }

    public String bookRoom(String name) {
        bookings.add(name);
        return "Room booked for " + name;
    }

    public String cancelBooking(String name) {
        if(bookings.remove(name)) {
            return "Booking cancelled for " + name;
        }
        return "No booking found";
    }
}
```

### 👉 Explanation:

* Stores bookings in list
* `bookRoom()` → adds name
* `cancelBooking()` → removes name

---

## 🔹 3. Server Main

```java
import java.rmi.Naming;

public class Server {
    public static void main(String[] args) {
        try {
            HotelImpl obj = new HotelImpl();
            Naming.rebind("HotelService", obj);
            System.out.println("Server is running...");
        } catch(Exception e) {
            System.out.println(e);
        }
    }
}
```

### 👉 Explanation:

* Creates server object
* Registers service using `Naming.rebind`
* Makes server available

---

## 🔹 4. Client Code

```java
import java.rmi.Naming;
import java.util.Scanner;

public class Client {
    public static void main(String[] args) {
        try {
            Hotel hotel = (Hotel) Naming.lookup("rmi://localhost/HotelService");

            Scanner sc = new Scanner(System.in);

            System.out.println("1. Book Room");
            System.out.println("2. Cancel Booking");

            int choice = sc.nextInt();
            sc.nextLine();

            System.out.print("Enter name: ");
            String name = sc.nextLine();

            if(choice == 1) {
                System.out.println(hotel.bookRoom(name));
            } else {
                System.out.println(hotel.cancelBooking(name));
            }

        } catch(Exception e) {
            System.out.println(e);
        }
    }
}
```

### 👉 Explanation:

* Connects to server
* Takes user input
* Calls remote methods
